# セットアップ手順書（Phase 1）

このドキュメント通りに進めれば、約30分でサイトが本番稼働します。

---

## 必要なもの

- サークル共有 Google アカウント（1個）
- GitHub アカウント（すでに `scicomi` あり）
- Webブラウザ

---

## STEP 1: スプレッドシートを作成（5分）

1. サークルGoogleアカウントで [Google スプレッドシート](https://docs.google.com/spreadsheets/) を開く
2. 「空白」から新規作成
3. ファイル名を **`SciComi_Portal_DB`** に変更
4. URLを見て **スプレッドシートID** をコピーしておく
   - 例：`https://docs.google.com/spreadsheets/d/【ここがID】/edit`
   - `1AbC...XyZ` のような長い文字列

> **注意**：シートは空のままでOK。次のSTEPでGASが自動でシートを作ります。

---

## STEP 2: GAS（Google Apps Script）を作成（10分）

### 2-1. プロジェクト作成
1. [script.google.com](https://script.google.com) を開く（同じサークルアカウントで）
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を **`SciComi_Portal_API`** に変更

### 2-2. コードを貼り付け
1. デフォルトの `function myFunction()` を全部消す
2. リポジトリの `gas/Code.gs` の中身を**全部コピペ**
3. 上の方の `SHEET_ID` を STEP 1 でコピーしたIDに書き換える
   ```js
   const SHEET_ID = '1AbC...XyZ'; // ← 実際のIDに置換
   ```
4. 💾 保存（Ctrl+S）

### 2-3. 初期セットアップを実行
1. 関数選択ドロップダウンで **`setupSpreadsheet`** を選ぶ
2. ▶ 実行 をクリック
3. **権限承認のダイアログが出る** → 「権限を確認」→ サークルアカウントを選択
4. 「このアプリは Google で確認されていません」→ **詳細** → **`SciComi_Portal_API` に移動（安全ではない）** をクリック
5. 「許可」をクリック
6. 実行ログに `Setup complete. Password: CHANGE_ME_xxxxx` と出れば成功

### 2-4. パスワードを変更（重要）
1. スプレッドシートを開き直して `Config` シートを確認
2. `password` の値が `CHANGE_ME_xxxxx` になっている → **覚えやすいパスワードに書き換える**
   - 例：`sc2026spring`
3. ここで決めたパスワードを後でメンバーに共有する

### 2-5. Webアプリとしてデプロイ
1. 右上の **デプロイ** → **新しいデプロイ**
2. 歯車アイコン⚙️ → **ウェブアプリ** を選択
3. 設定：
   - 説明：`v1`（任意）
   - 実行ユーザー：**自分**
   - アクセスできるユーザー：**全員**
4. **デプロイ** をクリック
5. **ウェブアプリのURL** が表示される → これをコピー
   - 形：`https://script.google.com/macros/s/AKfycb.../exec`

---

## STEP 3: フロントエンドにURLを設定（2分）

1. `api.js` を開く
2. 1行目付近の `API_URL` を STEP 2-5 のURLに書き換える
   ```js
   const API_URL = 'https://script.google.com/macros/s/AKfycb.../exec';
   ```
3. 保存

---

## STEP 4: GitHubにプッシュ＆動作確認（3分）

```bash
git add .
git commit -m "Phase 1: GAS バックエンド統合"
git push
```

数分後、`https://scicomi.github.io/portal-site/` にアクセス：

1. パスワード入力モーダルが出る ✅
2. STEP 2-4 で決めたパスワードを入力 → ログイン
3. 空のカレンダーが表示される ✅
4. 「＋ 新規イベント」でテスト追加
5. ページをリロード → イベントが残っている ✅ ← **これが成功の証**
6. スプレッドシートの `Events` シートを見ると、行が追加されている ✅

## トラブルシューティング

### ❌ ログインボタンを押しても何も起こらない
- ブラウザの開発者ツール（F12）→ コンソールタブでエラーを確認
- `Failed to fetch` → `API_URL` が間違っている、または GAS のデプロイ設定が「全員」になっていない
- `CORS error` → GAS再デプロイ時に「新しいデプロイ」ではなく既存デプロイの編集をしてしまった可能性。新規デプロイし直す

### ❌ 「unauthorized」と出る
- パスワードが間違っている
- Configシートを直接編集して `password` の値を確認
- ブラウザのコンソールで `localStorage.clear()` を実行してから再ログイン

### ❌ GASコードを修正したのに反映されない
- GASは **「新しいデプロイ」を作るか、デプロイ管理から「編集→新バージョン」して保存** が必要
- コードを変更しただけでは反映されない（Webアプリは固定バージョンを実行するため）

### ❌ スプレッドシートを直接編集しても、サイトに反映されない
- サイトはロード時にしかGASから取得しないので、リロードが必要
- Phase 3 で「定期更新」を実装予定
