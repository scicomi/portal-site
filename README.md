# SciComi Portal — サイエンスコミュニケーションサークル ポータルサイト

東海大学 サイエンスコミュニケーションサークル 向けのイベント管理ポータル。
カレンダー表示、イベント詳細管理、書類期限の自動計算、担当者管理などを行う。

**公開URL**: https://scicomi.github.io/portal-site/

## このサイトでできること

| 機能 | ページ | 説明 |
|---|---|---|
| ダッシュボード | ホーム | 直近イベント・期限・メンバー数の要約 |
| カレンダー表示 | イベント | FullCalendar月表示。祝日対応。ドラッグで新規作成 |
| イベント管理 | イベント | 4カテゴリ。検索・フィルタ・削除UNDO対応 |
| 書類期限の自動計算 | イベント | 許可願（-10日）・報告書（+7日）を自動算出 |
| メンバーリスト | メンバー | アドバイザー/コーディネーター/メンバーの3カテゴリ。テーブル表示 |
| 実験データベース | 実験内容 | 工作/実験ショー/その他。テーブル表示。物品・準備・流れ・注意事項 |
| パスワード認証 | 全ページ | サークル共通パスワード |
| データ永続化 | 全ページ | Google Sheets保存＋localStorageキャッシュで即表示 |
| リマインダーメール | 自動 | 期限の7/3/1日前にメール通知（GASトリガー） |
| 年間レポート | 手動/自動 | 年度ごとの活動レポートをメール送信 |

## システム構成

```
┌───────────────────┐    ┌─────────────────┐    ┌──────────────────┐
│  GitHub Pages     │ ⇄ │  Google Apps    │ ⇄ │  Google          │
│  (フロントエンド)  │    │  Script (API)   │    │  Sheets (DB)     │
│  HTML/CSS/JS      │    │  Code.gs        │    │  SciComi_Portal_DB│
└───────────────────┘    └─────────────────┘    └──────────────────┘
        ↑                                                 ↑
   メンバーがアクセス                                  幹部が直接編集も可
```

| レイヤー | サービス | 役割 |
|---|---|---|
| **フロントエンド** | GitHub Pages | 静的HTML/CSS/JSを配信。無料・自動デプロイ |
| **バックエンドAPI** | Google Apps Script (GAS) | REST APIとして動作。CRUD操作・認証 |
| **データベース** | Google Sheets | イベント情報を保存。幹部が直接スプレッドシート編集も可能 |

### なぜこの構成か

- **無料**：サークル予算ゼロで運用可能
- **専門知識不要**：サーバー管理・データベース管理が一切不要
- **代替手段あり**：万一サイトが壊れてもスプレッドシートを直接見れば情報は失われない
- **既存ツールとの親和性**：サークルメンバーは全員Googleアカウントを持っている

## ページ構成

ポータルは4ページ構成：

| ページ | URL | 役割 |
|---|---|---|
| ホーム | `index.html` | ダッシュボード。直近イベント・期限・メンバー数等の要約 |
| イベント | `events.html` | カレンダー＋イベント管理（CRUD） |
| メンバー | `members.html` | 3カテゴリ（アドバイザー/コーディネーター/メンバー）テーブル表示 |
| 実験内容 | `experiments.html` | フィルタ式（工作/実験ショー/その他）の実験データベース |

## ファイル構成

```
portal-site/
├── index.html         # ホーム（ダッシュボード）
├── events.html        # イベント管理
├── members.html       # メンバーリスト
├── experiments.html   # 実験内容
├── style.css          # 共通スタイル
├── config.js          # ★一元設定（URL・カテゴリ定義・期限ルール等）
├── api.js             # GAS通信レイヤー（3リソース対応）
├── app.js             # 共通ロジック（認証/ナビ/同期/トースト/ユーティリティ）
├── home.js            # ホームページ
├── script.js          # イベントページ
├── members.js         # メンバーページ
├── experiments.js     # 実験ページ
├── README.md
├── gas/
│   └── Code.gs        # GASに貼り付けるバックエンド（3リソース対応）
└── docs/
    ├── 01_sheets_schema.md
    ├── 02_setup_guide.md
    └── 03_phase3_proposal.md
```

> **スクリプトの読み込み順**は全ページ共通で
> `config.js → api.js → app.js → 各ページ.js`。
> config.js が他の全てより先に読まれる前提なので、この順序は変えないこと。

### 拡張するときはまず config.js

カテゴリ追加・色変更・URL変更・期限ルール変更は、原則 **config.js だけ** 直せば全ページに反映される設計です。

| やりたいこと | config.js の該当箇所 |
|---|---|
| GASのURLを変えた | `API_URL` |
| イベント種類を増やす | `EVENT_CATEGORIES` に1行 |
| 実験タブを増やす | `EXPERIMENT_CATEGORIES` に1行 |
| メンバー区分を増やす | `MEMBER_CATEGORIES` に1行 |
| 書類期限の日数を変える | `DEADLINE_RULES` |
| 期限警告の色しきい値 | `DEADLINE_ALERT` |
| リマインダー通知日数 | `REMINDER.days` |

### 各ファイルの役割

| ファイル | 役割 | 編集頻度 |
|---|---|---|
| `index.html` 等HTML | 各ページの構造 | UI追加時 |
| `style.css` | 全ページ共通スタイル | デザイン変更時 |
| `app.js` | 認証・ヘッダー・同期・トースト（全ページが読み込む） | 共通機能追加時 |
| `api.js` | GASとの通信。3リソース（events/members/experiments）対応 | **API_URL更新時のみ** |
| `script.js` | イベント管理ロジック | イベント機能追加時 |
| `members.js` | メンバー管理 | メンバー機能追加時 |
| `experiments.js` | 実験管理 | 実験機能追加時 |
| `home.js` | ダッシュボード | ホーム表示変更時 |
| `gas/Code.gs` | バックエンドAPI | **GASエディタで管理（GitHubは履歴用）** |

## 開発・編集方法

### 1. ローカルで編集する

```bash
git clone https://github.com/scicomi/portal-site.git
cd portal-site
# お好みのエディタで編集（VSCode推奨）
```

### 2. ローカルで動作確認する

Webブラウザで `index.html` を直接開いてもCORSエラーで動かないので、簡易サーバーを立てる：

```bash
# Python 3 がインストールされている場合
python -m http.server 8000

# または Node.js
npx serve
```

ブラウザで `http://localhost:8000` を開く。

### 3. GitHubにプッシュ＝自動的に本番反映

```bash
git add .
git commit -m "変更内容の説明"
git push
```

数分後に https://scicomi.github.io/portal-site/ に自動反映される。

### 4. GAS（バックエンド）を編集する場合

1. [script.google.com](https://script.google.com) を開く
2. プロジェクト `SciComi_Portal_API` を開く
3. `Code.gs` を編集
4. 保存
5. **デプロイ → デプロイを管理 → 編集 → バージョン: 新しいバージョン → デプロイ**

**重要**：GASは「保存」だけでは公開されない。**必ず「新しいバージョン」でデプロイ**すること。

### 5. GitHubのコードとGASのコードを同期する

GASエディタでは `gas/Code.gs` の内容をコピペで使う。GitHubのリポジトリにある `gas/Code.gs` は**履歴管理用**であり、GASエディタとは自動連携していない。手作業で同期する。

## 運用・保守

### パスワードを変更したい

1. スプレッドシート `SciComi_Portal_DB` を開く
2. `Config` シート → `password` の値を新しいものに変更
3. メンバーに新パスワードを共有
4. 各メンバーは次回ログイン時に新パスワードを入力すればOK

### 過去のイベントを直接修正したい

スプレッドシートの `Events` シートを直接編集してOK。
ただし以下のフィールドは特殊なので注意：
- `PartsList`：JSON文字列。形式を壊さないこと
- `Files`：JSON文字列。同上
- `ID`：絶対に変更しない

### イベントをCSVでエクスポートしたい

スプレッドシート `Events` シートで **ファイル → ダウンロード → カンマ区切り形式(.csv)**

### ストレージ容量を確認したい

スプレッドシートには事実上容量制限なし（500万セルまで）。
Google Drive（将来Phase 2でファイル保存に使う）の容量は drive.google.com で確認。

### Phase 4: リマインダーメール・年間レポート

**リマインダーメール（毎日自動）**

イベントの許可願・報告書期限が近づくと、担当者とアドバイザー/コーディネーターに自動メールが届く。

セットアップ（1回だけ）:
1. GASエディタで `installTriggers()` を実行
2. Config シートで `reminder_enabled` が `true` になっていることを確認
3. `reminder_days` で通知日数を調整（デフォルト: 7日前, 3日前, 1日前）
4. メンバーページでアドバイザー/コーディネーターのメールアドレスを登録

**年間レポート**

年度の活動レポート（イベント数・実験使用ランキング・メンバー統計）をHTML形式でメール送信。

GASエディタで `generateAnnualReport(2026)` を実行（引数なしなら現在年度を自動判定）。
送信先は Config シートの `report_recipients`（空欄ならアドバイザー/コーディネーター全員）。

### バックアップを取る

月1回程度、以下を推奨：
1. スプレッドシート → ファイル → コピーを作成
2. ファイル名に日付を入れる：`SciComi_Portal_DB_backup_2026-06`

## トラブルシューティング

### ログインしても何も起こらない

ブラウザで F12 → Console タブを見る：

```js
// 現在のトークンを確認
localStorage.getItem('scicomi_portal_token')
```

→ `null` → ログインに失敗している。パスワードを再確認。
→ 何かの文字列 → トークンは保存済み。GAS側の問題の可能性。

### 「unauthorized」エラーが出る

```js
// localStorage の値とスプレッドシート Config の password が一致しているか確認
console.log(localStorage.getItem('scicomi_portal_token'));
// → スプレッドシートで Config > password の値と比較
// → 違ったら以下で初期化して再ログイン
localStorage.clear();
location.reload();
```

### 「Failed to fetch」エラー

- `api.js` の `API_URL` がGASのデプロイURLと一致しているか確認
- GASのデプロイ設定が「アクセスできるユーザー: 全員」になっているか確認

### 「CORS policy」エラー

GAS Web Appは正常な200レスポンス時のみCORSヘッダーを付ける。
**実体は GAS が 500 エラーを返している**ことが多い。

1. GASエディタを開く
2. 左メニュー「実行数」を開く
3. 直近の❌（失敗）の詳細を見る
4. エラーメッセージから原因を特定

### GASのコード変更が反映されない

GASは保存だけでは公開されない。
**デプロイ → デプロイを管理 → ✏️ → バージョン: 新しいバージョン → デプロイ** が必要。

### スプレッドシートを編集したのにサイトに反映されない

サイトはロード時にしかGASから取得しない。**ブラウザをリロード**すれば反映される。
Phase 3 で自動リフレッシュ実装予定。

## 引き継ぎ情報

### 重要な認証情報の管理

以下の情報は **新幹部に必ず引き継ぐ** こと：

1. **サークル用Googleアカウント** のID/パスワード
   - スプレッドシート、GAS、GitHub Pages 設定に必要
2. **GitHub Organization** (`scicomi`) のオーナー権限
   - 設定 → People → 新メンバーをOwnerに昇格
3. **ポータルのログインパスワード**
   - スプレッドシート `Config` シート → `password`

### 引き継ぎ前のチェックリスト

```
☐ Googleアカウントのパスワードを新幹部に共有
☐ GitHub Organization のOwner権限を新幹部に付与
☐ GASプロジェクトの編集者権限を新幹部に共有
   - script.google.com で「共有」→ 新幹部のメールアドレス追加
☐ スプレッドシートの編集権限を新幹部に共有
☐ 新幹部にこのREADMEを読んでもらう
☐ 新幹部と一緒にイベント追加・編集をテスト
```

### よくある質問

**Q. 開発者が引退した後にバグが出たらどうする？**

A. 主な選択肢：
- スプレッドシートは無傷なので、最悪サイトが壊れてもデータは失われない
- 知り合いのエンジニアに依頼する（このREADMEを渡せば1日で理解可能）
- 復旧諦めてスプレッドシート直接運用に戻る（ポータルの本質的な意義は失われるが、データは保持される）

**Q. サイトをまるごと別の場所に移したい**

A. このリポジトリをフォークして、`api.js` の `API_URL` を新しいGASに向ければOK。
HTML/CSS/JS は完全に静的なので、Netlify、Vercel、Cloudflare Pages など好きなホスティングに移せる。

**Q. データだけ取り出したい**

A. スプレッドシートを直接開いてCSVエクスポート。技術力ゼロでも可能。
